# README Generation Skill

## Purpose

This skill is responsible for generating professional,
structured, and production-oriented README documentation
for data engineering and cloud engineering projects.

The goal is to create clear project documentation
that explains architecture, implementation,
technical decisions, and operational workflows.

---

# Responsibilities

The skill should assist with:

* README generation
* Project overview creation
* Architecture summaries
* Technology stack documentation
* AWS service descriptions
* Data flow explanations
* Installation instructions
* Setup guidance
* Troubleshooting sections
* Lessons learned documentation
* Technical decision summaries
* Future improvement sections
* Visualization documentation
* Repository structure explanations

---

# Preferred Outputs

Outputs should include:

* clean Markdown formatting
* professional documentation structure
* business-friendly explanations
* technical implementation details
* architecture context
* operational guidance
* implementation-oriented descriptions
* reusable README patterns

---

# Engineering Principles

The skill should:

* prioritize readability
* avoid unnecessary verbosity
* explain technical decisions clearly
* support portfolio-quality documentation
* encourage structured storytelling
* document real engineering challenges
* highlight architectural tradeoffs
* support onboarding and maintainability

---

# Common Use Cases

Examples include:

* AWS streaming projects
* OpenSearch analytics platforms
* ETL pipelines
* Data lake architectures
* Real-time monitoring systems
* Event-driven systems
* Lambda architectures
* Athena analytics workflows
* Portfolio projects
* Production engineering repositories
