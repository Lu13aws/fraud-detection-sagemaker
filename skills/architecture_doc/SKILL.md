# Architecture Documentation Skill

## Purpose

This skill is responsible for generating, reviewing,
and maintaining architecture documentation for data engineering,
cloud, analytics, and streaming systems.

The skill should help create clear, maintainable,
and business-friendly architecture explanations.

---

# Responsibilities

The skill should assist with:

* Architecture diagram generation
* Data flow documentation
* AWS service interaction explanations
* Event-driven architecture documentation
* Streaming pipeline documentation
* Infrastructure component descriptions
* System boundary definitions
* Hot vs cold path explanations
* Security and IAM documentation
* Operational workflow explanations
* Technical decision documentation
* Architecture tradeoff analysis

---

# Preferred Outputs

Outputs should include:

* clear architecture explanations
* structured data flows
* readable technical documentation
* practical operational context
* business-friendly summaries
* implementation-oriented diagrams
* modular system descriptions

---

# Engineering Principles

The skill should:

* prioritize clarity over complexity
* avoid unnecessary architectural buzzwords
* explain service interactions clearly
* document tradeoffs transparently
* support maintainability and onboarding
* encourage modular architectures
* emphasize operational simplicity

---

# Common Use Cases

Examples include:

* AWS streaming architectures
* Lambda event pipelines
* OpenSearch analytics systems
* Kinesis data flows
* Data lake architectures
* ETL and ELT systems
* Event-driven systems
* Real-time analytics platforms
* Monitoring architectures
