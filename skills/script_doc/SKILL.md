# Script Documentation Skill

## Purpose

This skill is responsible for documenting scripts,
automation utilities, helper tools, and operational workflows
used in data engineering projects.

The goal is to improve maintainability,
operational transparency, and reusability.

---

# Responsibilities

The skill should assist with:

* Script purpose documentation
* Input and output explanations
* Usage instructions
* Environment requirement documentation
* Dependency explanations
* Execution flow descriptions
* Side effect identification
* Risk and safety documentation
* Automation workflow explanations
* Troubleshooting guidance
* Operational notes
* Validation step documentation

---

# Preferred Outputs

Outputs should include:

* concise script summaries with bullet points
* execution examples
* parameter explanations
* environment setup requirements
* operational warnings where appropriate
* expected outputs
* troubleshooting notes
* reusable documentation patterns

---

# Engineering Principles

The skill should:

* prioritize operational clarity
* clearly identify side effects
* document potentially dangerous operations
* encourage reusable scripts
* minimize ambiguity
* support maintainability and onboarding
* separate setup logic from execution logic

---

# Common Use Cases

Examples include:

* project bootstrap scripts
* environment setup scripts
* deployment scripts
* infrastructure automation scripts
* validation scripts
* data ingestion utilities
* OpenSearch setup scripts
* Athena helper scripts
* Lambda deployment utilities
* monitoring scripts
* troubleshooting utilities
